# 函数式接口 (Functional Interfaces)

## 1. 核心概念与理论基础

### 1.1 定义 (Definition)
**函数式接口 (Functional Interface)** 是 Java 8 引入的核心概念，它是 Lambda 表达式和方法引用的**目标类型**。
*   **严格定义**：接口中**有且仅有一个抽象方法** (Single Abstract Method, SAM)。
*   **允许存在**：
    *   任意数量的 `default` 方法 (默认方法)。
    *   任意数量的 `static` 方法 (静态方法)。
    *   从 `java.lang.Object` 继承的方法 (如 `equals`, `hashCode`, `toString`)，这些**不计入**抽象方法。

### 1.2 @FunctionalInterface 注解
*   **位置**：标注在接口定义上。
*   **作用**：
    1.  **编译期检查**：如果接口不满足函数式接口定义（例如增加了第二个抽象方法），编译器直接报错。
    2.  **文档说明**：明确告知开发者该接口设计意图是用于 Lambda 表达式。
*   **注意**：即使不加该注解，只要满足 SAM 原则，接口依然是函数式接口。但**强烈建议加上**。

```java
// 正确示例
@FunctionalInterface
interface ValidFunc {
    void exec(); 
    default void log() {} // 允许
}

// 错误示例 (编译报错)
@FunctionalInterface
interface InvalidFunc {
    void exec();
    void run(); // 两个抽象方法，报错
}
```

### 1.3 底层原理 ( invokedynamic )
*   Java 8 之前：匿名内部类会在编译时生成独立的 `.class` 文件。
*   Java 8 之后：Lambda 表达式通过 `invokedynamic` 字节码指令实现。
    *   **优势**：运行时动态链接，减少了类的生成数量，提高了性能，且能更好地访问外部变量。
    *   **实现**：JVM 在运行时通过 `LambdaMetafactory` 动态生成实现类的实例。

## 2. java.util.function 包全景图谱

Java 8 在 `java.util.function` 包中提供了 40+ 个预定义接口。为了体系化记忆，我们将其分为四类。

### 2.1 分类逻辑表

| 分类 | 接口名 | 泛型 | 抽象方法 | 输入参数 | 返回值 | 典型用途 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **供给型** | `Supplier<T>` | `<T>` | `T get()` | 0 | 1 (T) | 工厂、延迟加载、生成数据 |
| **消费型** | `Consumer<T>` | `<T>` | `void accept(T)` | 1 (T) | 0 (void) | 打印、发送、修改状态 |
| **函数型** | `Function<T,R>` | `<T,R>` | `R apply(T)` | 1 (T) | 1 (R) | 类型转换、映射、计算 |
| **断言型** | `Predicate<T>` | `<T>` | `boolean test(T)` | 1 (T) | 1 (boolean) | 过滤、校验、条件判断 |
| **双参型** | `BiFunction<T,U,R>` | `<T,U,R>` | `R apply(T,U)` | 2 (T,U) | 1 (R) | 多参数计算 |
| | `BiConsumer<T,U>` | `<T,U>` | `void accept(T,U)` | 2 (T,U) | 0 | 多参数消费 |
| | `BiPredicate<T,U>` | `<T,U>` | `boolean test(T,U)` | 2 (T,U) | 1 (boolean) | 多参数校验 |
| **同类型** | `UnaryOperator<T>` | `<T>` | `T apply(T)` | 1 (T) | 1 (T) | 输入输出类型一致 (Function 特化) |
| | `BinaryOperator<T>` | `<T>` | `T apply(T,T)` | 2 (T,T) | 1 (T) | 两个同类型输入 (BiFunction 特化) |
| **基础型** | `Runnable` | 无 | `void run()` | 0 | 0 | 线程任务 |
| | `Callable<V>` | `<V>` | `V call()` | 0 | 1 (V) | 线程任务 (可抛异常) |
| | `Comparator<T>` | `<T>` | `int compare(T,T)` | 2 (T,T) | 1 (int) | 比较排序 |

### 2.2 记忆心法
*   **看前缀**：`Bi` 代表两个参数。
*   **看后缀**：
    *   `Supplier` -> 供给 (无进有出)
    *   `Consumer` -> 消费 (有进无出)
    *   `Function` -> 函数 (有进有出)
    *   `Predicate` -> 断言 (有进布尔出)
    *   `Operator` -> 运算 (输入输出类型相同)

## 3. 四大核心接口深度解析
### 3.1 Supplier<T> (供给型)
*   **核心方法**：`T get()`
*   **特性**：无参数，依赖外部上下文或内部逻辑产生结果。
*   **默认方法**：无。
*   **应用场景**：
    1.  **工厂模式**：`Supplier<User> userFactory = () -> new User();`
    2.  **Optional 懒加载**：`optional.orElseGet(() -> getDefaultValue());`
    3.  **Stream 生成**：`Stream.generate(() -> Math.random()).limit(5);`

### 3.2 Consumer<T> (消费型)
*   **核心方法**：`void accept(T t)`
*   **特性**：侧重副作用 (Side Effect)，如打印、写入数据库。
*   **默认方法**：
    *   `andThen(Consumer<? super T> after)`：链式消费。
*   **代码示例**：
```java
Consumer<String> c1 = s -> System.out.print(s + " ");
Consumer<String> c2 = s -> System.out.println(s.toUpperCase());

// 先执行 c1，再执行 c2
c1.andThen(c2).accept("hello"); 
// 输出：hello HELLO
```

### 3.3 Function<T, R> (函数型)
*   **核心方法**：`R apply(T t)`
*   **特性**：数据转换的核心，输入 T 经过处理变为 R。
*   **默认方法**：
    *   `andThen(Function<? super R, ? extends V> after)`：先当前，后 after。 `(f andThen g)(x) = g(f(x))`
    *   `compose(Function<? super V, ? extends T> before)`：先 before，后当前。 `(f compose g)(x) = f(g(x))`
*   **代码示例**：
```java
Function<Integer, Integer> add1 = x -> x + 1;
Function<Integer, Integer> mul2 = x -> x * 2;

// andThen: 先加 1，再乘 2 -> (1+1)*2 = 4
System.out.println(add1.andThen(mul2).apply(1)); 

// compose: 先乘 2，再加 1 -> (1*2)+1 = 3
System.out.println(add1.compose(mul2).apply(1)); 
```

### 3.4 Predicate<T> (断言型)
*   **核心方法**：`boolean test(T t)`
*   **特性**：返回布尔值，用于逻辑判断。
*   **默认方法**：
    *   `and`, `or`, `negate` (逻辑与、或、非)。
*   **应用场景**：`Stream.filter()`, `Collection.removeIf()`。
*   **代码示例**：
```java
Predicate<Integer> isPositive = x -> x > 0;
Predicate<Integer> isEven = x -> x % 2 == 0;

// 组合条件：大于 0 且 为偶数
Predicate<Integer> complex = isPositive.and(isEven);
List<Integer> result = list.stream().filter(complex).collect(Collectors.toList());
```

## 4. 基本类型特化接口 (性能关键)

**为什么需要特化接口？**
泛型在 Java 中会发生**类型擦除**，且泛型不支持基本类型 (`int`, `double` 等)。如果使用 `Function<Integer, Integer>`，会发生 **装箱 (Boxing)** 和 **拆箱 (Unboxing)**，在大数据量循环中会造成严重的性能损耗和 GC 压力。

**解决方案**：`java.util.function` 提供了针对 `int`, `long`, `double` 的特化接口。

### 4.1 常见特化接口表

| 原始接口 | int 特化 | long 特化 | double 特化 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `Supplier<R>` | `IntSupplier` | `LongSupplier` | `DoubleSupplier` | 返回基本类型 |
| `Consumer<T>` | `IntConsumer` | `LongConsumer` | `DoubleConsumer` | 接受基本类型 |
| `Function<T,R>` | `IntFunction<R>` | `LongFunction<R>` | `DoubleFunction<R>` | 输入基本，输出对象 |
| `Function<T,R>` | `ToIntFunction<T>` | `ToLongFunction<T>` | `ToDoubleFunction<T>` | 输入对象，输出基本 |
| `Predicate<T>` | `IntPredicate` | `LongPredicate` | `DoublePredicate` | 判断基本类型 |
| `UnaryOperator<T>`| `IntUnaryOperator`| `LongUnaryOperator`| `DoubleUnaryOperator`| 基本类型转基本类型 |
| `BinaryOperator<T>`| `IntBinaryOperator`| `LongBinaryOperator`| `DoubleBinaryOperator`| 两个基本转一个基本 |

### 4.2 性能对比示例
```java
// ❌ 低效：涉及 Integer 装箱拆箱
Stream<Integer> stream = Stream.of(1, 2, 3);
int sum1 = stream.mapToInt(i -> i * 2).sum(); 

// ✅ 高效：直接使用 IntStream
IntStream intStream = IntStream.of(1, 2, 3);
int sum2 = intStream.map(i -> i * 2).sum(); 
```
**笔记重点**：在处理数值计算、统计、大规模集合遍历是，**优先使用** `IntStream`, `LongStream`, `DoubleStream` 及其对应的函数式接口。

## 5. Lambda 与方法引用机制

### 5.1 Lambda 表达式语法糖
Lambda 是匿名函数的简洁表示。
```java
// 1. 无参数
() -> System.out.println("Hello");

// 2. 单参数 (可省略括号)
x -> x * x

// 3. 多参数
(x, y) -> x + y

// 4. 代码块 (需显式 return)
(x, y) -> {
    int sum = x + y;
    return sum;
}
```

### 5.2 方法引用 (Method Reference)
当 Lambda 体仅仅是调用一个已有方法时，使用方法引用 `::` 更简洁。

| 类型 | Lambda 写法 | 方法引用写法 | 说明 |
| :--- | :--- | :--- | :--- |
| **静态方法** | `x -> Integer.parseInt(x)` | `Integer::parseInt` | 类名::静态方法 |
| **实例方法** | `str -> str.length()` | `String::length` | 类名::实例方法 (第一个参数作为调用对象) |
| **特定对象** | `x -> object.method(x)` | `object::method` | 实例::实例方法 |
| **构造器** | `s -> new String(s)` | `String::new` | 类名::new |
| **数组构造** | `len -> new int[len]` | `int[]::new` | 类型 []::new |

### 5.3 变量捕获规则 (Effectively Final)
Lambda 表达式可以访问外部局部变量，但该变量必须是 **final** 或 **effectively final** (初始化后未被修改)。
*   **原因**：Lambda 可能异步执行，如果变量可变，会导致线程安全问题或数据不一致。
*   **错误示例**：
```java
int count = 0;
Runnable r = () -> {
    count++; // 编译报错：Variable used in lambda expression should be final or effectively final
};
```
## 6. 高级场景与常见陷阱

### 6.1 受检异常 (Checked Exception) 处理
标准函数式接口（如 `Function`, `Consumer`）的方法签名**不声明抛出受检异常**。
*   **问题**：如果在 Lambda 中调用可能抛出 `IOException` 的方法，无法直接抛出。
*   **解决方案**：
    1.  **Try-Catch 包裹**：在 Lambda 内部处理。
    2.  **自定义接口**：定义允许抛异常的函数式接口。
    3.  **包装工具类**：将受检异常包装为运行时异常。

```java
// 方案 2：自定义接口
@FunctionalInterface
interface IOFunction<T, R> {
    R apply(T t) throws IOException;
}

// 方案 3：工具类包装
public static <T, R> Function<T, R> wrapper(CheckedFunction<T, R> func) {
    return t -> {
        try { return func.apply(t); }
        catch (Exception e) { throw new RuntimeException(e); }
    };
}
```

### 6.2 Lambda 的序列化
*   Lambda 表达式实现的类是动态生成的，类名不确定 (如 `Main$$Lambda$1/0x0000000800b79440`)。
*   **结论**：**Lambda 表达式通常不建议直接序列化**。如果接口需要序列化，应使用匿名内部类或显式实现类。
*   `Serializable` 特化：部分接口如 `Serializable` 可以配合使用，但需谨慎。

### 6.3 调试困难
*   Lambda 在堆栈跟踪中显示为 `lambda$main$0` 等形式，难以定位具体代码行。
*   **建议**：复杂的 Lambda 逻辑应提取为独立的私有方法，然后使用方法引用。
```java
// ❌ 难调试
list.stream().filter(x -> {
    // 复杂逻辑...
    return x > 10;
});

// ✅ 易调试
list.stream().filter(this::isGreaterThanTen); 

private boolean isGreaterThanTen(Integer x) {
    return x > 10;
}
```

### 6.4 空指针风险
*   在使用 `Function.andThen` 或 `Comparator` 时，如果中间结果为 `null`，可能引发 NPE。
*   **建议**：使用 `Objects.requireNonNull` 或 `Optional` 进行防御性编程。

## 7. 自定义函数式接口

虽然 `java.util.function` 覆盖了 90% 的场景，但在以下情况需自定义：
1.  **需要抛出受检异常** (见 6.1)。
2.  **语义更明确**：例如定义 `Validator<T>` 比 `Predicate<T>` 业务含义更清晰。
3.  **特殊泛型约束**：需要限定泛型继承关系。

**示例：带异常处理的消费者**
```java
@FunctionalInterface
public interface ThrowingConsumer<T, E extends Exception> {
    void accept(T t) throws E;
}

// 使用
public void processList(List<String> list) {
    ThrowingConsumer<String, IOException> writer = s -> {
        // 可能抛 IOException 的写入操作
        Files.write(Paths.get("log.txt"), s.getBytes()); 
    };
    
    for (String s : list) {
        try {
            writer.accept(s);
        } catch (Exception e) {
            // 统一异常处理
            log.error("Write failed", e);
        }
    }
}
```
## 8. 总结与面试速查

### 8.1 高频面试题 (Q&A)
1.  **Q: 什么是函数式接口？如何定义？**
    *   A: 只有一个抽象方法的接口。使用 `@FunctionalInterface` 注解标记。
2.  **Q: `Function` 和 `Consumer` 的区别？**
    *   A: `Function` 有返回值 (用于转换)，`Consumer` 无返回值 (用于消费/副作用)。
3.  **Q: 为什么需要 `IntFunction` 这种基本类型接口？**
    *   A: 避免泛型装箱拆箱带来的性能损耗和内存开销。
4.  **Q: Lambda 表达式中访问局部变量有什么限制？**
    *   A: 变量必须是 final 或 effectively final (不可变)。
5.  **Q: 接口中有 default 方法还是函数式接口吗？**
    *   A: 是，只要抽象方法只有一个，default 方法不计入。

### 8.2 最佳实践建议
1.  **优先使用标准接口**：不要重复造轮子，优先查 `java.util.function` 表。
2.  **数值计算用特化**：涉及 `int/long/double` 计算，务必用 `IntStream` 等。
3.  **保持简洁**：Lambda 体超过 3 行，建议提取为方法引用。
4.  **注意副作用**：在 `Stream` 操作中，尽量使用无状态、无副作用的 Lambda，便于并行处理。
